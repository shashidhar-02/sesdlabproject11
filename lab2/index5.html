Proper Use of Labels with Form Inputs

Why Labels Matter

  • Screen Readers: Labels are announced to users with visual impairments
  • Click Target: Clicking the label focuses the input (larger click area)
  • Clarity: Makes forms easier to understand for all users
  • Accessibility: Essential for WCAG 2.1 compliance

Images with Meaningful Alt Attributes

Importance of Alt Text

  • Screen Readers: Alt text is read aloud to users who can't see the image
  • SEO: Search engines use alt text to understand image content
  • Broken Images: Alt text displays when images fail to load
  • Context: Should describe the image's purpose, not just its appearance
  • Decorative Images: Use empty alt="" to skip unnecessary descriptions

Informative Alt Text

Alt: "Software developer typing code on a laptop with multiple monitors displaying colorful syntax highlighting"

Describes what's happening in the image and provides context.

Contextual Alt Text

Alt: "Diverse team of developers collaborating around a whiteboard with flowcharts and diagrams"

Provides meaningful context about the team collaboration scene.

Accessibility Focus

Alt: "Person using assistive technology with screen reader to navigate a website"

Highlights the importance of accessible web design.

Best Practices for Alt Text

  • Be specific and concise (aim for under 125 characters)
  • Don't start with "Image of..." or "Picture of..."
  • Include text that appears in the image
  • Describe the function, not just the appearance (e.g., for buttons)
  • Use alt="" for purely decorative images
  • For complex images (charts, diagrams), provide detailed description nearby

Accessibility Best Practices Demonstrated

✓ Semantic HTML

  • Used <header>, <nav>, <main>, <section>, <footer>
  • Proper heading hierarchy (h1 → h2 → h3)
  • Semantic form elements with <fieldset> and <legend>

✓ Keyboard Navigation

  • Tab Order: All interactive elements are keyboard accessible
  • Focus Indicators: Clear visual feedback when elements are focused
  • Skip Link: "Skip to main content" for keyboard users
  • Logical Flow: Content flows in a meaningful order

✓ ARIA (Accessible Rich Internet Applications)

  • role attributes (banner, navigation, main, complementary)
  • aria-label and aria-labelledby for context
  • aria-required for required form fields
  • aria-describedby to link hints with inputs
  • aria-current="page" for navigation state

✓ Forms Accessibility

  • Every input has a properly associated <label> using for attribute
  • Required fields clearly marked with visual and screen reader indicators
  • Helpful hints linked to inputs via aria-describedby
  • Autocomplete attributes for easier form filling
  • Grouped related inputs with <fieldset>

✓ Images Accessibility

  • All images have meaningful alt attributes
  • Alt text describes content and context, not just appearance
  • Lazy loading for performance: loading="lazy"
  • Responsive images that scale properly

Testing Your Accessibility

Keyboard Navigation Test:

  1. Press Tab to navigate forward through interactive elements
  2. Press Shift + Tab to navigate backward
  3. Press Enter or Space to activate buttons/links
  4. Verify all interactive elements have visible focus indicators

Screen Reader Test: Use NVDA (Windows), JAWS, or VoiceOver (Mac) to test how content is announced.
